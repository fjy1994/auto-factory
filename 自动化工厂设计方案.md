# 自动化工厂设计方案

> 最后更新: 2026-05-28
> 本文档描述系统的整体架构和业务流程，不涉及具体代码实现和技术细节。

---

## 1. 系统概述

自动化工厂是一套面向 ROM 分支的自动化测试管理系统，覆盖从 ROM 版本跟踪、刷机部署、分支任务执行到测试结果汇总的完整流程。系统围绕"ROM 分支"和"刷机-测试"流水线设计，帮助团队高效管理多设备、多分支的测试覆盖。

### 核心业务流程

1. ROM 构建后发送邮件通知 → 系统自动/手动录入版本跟踪
2. 对匹配分支的版本触发刷机流程（下载ROM → 刷机 → 开机引导 → 登录WiFi）
3. 刷机成功后按分支配置顺序执行任务（调用用例集/脚本）
4. 测试结果回填到版本跟踪，展示覆盖率与通过率

---

## 2. 系统架构

系统采用三层架构：

- **前端管理界面** — 提供 Web 管理操作界面，涵盖仪表盘、设备管理、分支管理、刷机管理、版本跟踪、任务中心、用例管理等模块
- **后端服务** — 提供 REST API 和 WebSocket 通信，负责业务逻辑处理、数据存储和刷机状态推送
- **设备 Agent** — 部署在执行机上，通过 ADB 控制 Android 设备，执行刷机、开机引导、WiFi 登录等操作

三层之间通过 HTTP API 和 WebSocket 进行通信。

## 3. 各模块功能说明

### 3.1 仪表盘 (Dashboard)

通过图表展示系统整体概况：设备状态分布、任务执行趋势、版本覆盖概览。

### 3.2 设备管理 (Device)

管理接入的 Android 设备，记录设备序列号、机型、ROM 版本、当前状态（空闲/忙碌/离线/异常）等信息。支持卡片面布局和分页。

### 3.3 分支管理 (Branch)

管理 ROM 分支。分支按类型分为 DEV / 主干 / 商分三类，每个分支可配置版本匹配规则和邮件标题匹配规则（支持正则）。分支可禁用/启用。

分支展开后可配置**任务配置**，用于定义刷机成功后按顺序执行的自动化任务（关联用例集或脚本）。

### 3.4 刷机管理 (Flashing)

管理每个设备的刷机过程。每个刷机过程包含4个标准步骤：

1. **下载 ROM** — 从指定地址下载 ROM 包
2. **刷机** — 通过 ADB/刷机工具将 ROM 写入设备
3. **开机引导** — 设备重启后的初始设置引导
4. **登录 WiFi** — 配置网络连接

每个步骤独立管理状态，支持失败后手动触发单步重试或全部重试。刷机过程卡片以时间线形式展示各步骤状态。

### 3.5 版本跟踪 (RomTrack)

按 ROM 分支分组展示各版本的测试覆盖情况。顶部统计面板展示已覆盖 / 覆盖中 / 未覆盖数量。每个版本卡片展示测试结果（通过率、通过数、失败数）以及未覆盖原因分析（无设备、刷机失败等）。

版本跟踪与刷机管理和任务中心联动：刷机失败记录可直接跳转到刷机管理页面定位到对应刷机过程；任务配置行可直接跳转到任务中心定位到对应任务。

### 3.6 任务中心 (Task)

展示刷机成功后触发的执行任务，支持按状态和分支筛选。展示任务关联的分支、版本、机型、执行进度、通过/失败统计。

### 3.7 用例管理 (TestCase)

管理测试用例库。用例按 L0~L4 五级优先级分类，支持按模块和优先级筛选。提供 Excel 批量导入功能，导入时自动识别表头并智能映射优先级。

## 4. 关键设计决策

### 4.1 ROM 分支概念

分支是围绕 ROM 版本管理的核心实体，按 DEV / 主干 / 商分 + 版本号区分，不含应用名称或 APK 名称概念。版本号匹配和邮件标题匹配支持正则表达式，用于自动关联构建通知到对应分支。

### 4.2 Mock 数据优先

前端默认使用 Mock 数据运行，确保后端未就绪时 UI 可独立开发和演示。用户可通过开关随时切换到真实后端数据。

### 4.3 刷机过程独立管理

刷机作为一个独立实体管理，不在任务中内嵌。支持独立的状态管理和逐步骤重试，每个步骤的错误信息、重试次数、当前状态独立记录。

### 4.4 版本跟踪聚焦测试覆盖

版本跟踪页面以测试覆盖率为核心指标，展示已覆盖/覆盖中/未覆盖的统计，以及未覆盖原因分析（无设备、刷机失败等），而非聚焦刷机状态。

### 4.5 跨页面跳转聚焦

多个页面支持通过焦点参数实现跨页面定位。例如版本跟踪中的刷机失败可跳转到刷机管理并高亮对应卡片，任务配置行可跳转到任务中心并定位到目标任务。

### 4.6 数据流

```
前端页面 → 状态管理层 → 数据请求层 → 后端 REST API
                           ↕
                     Mock 数据（可选）
```
